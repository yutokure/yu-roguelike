1. マニュアルの読み方

このマニュアルは、ゲームを遊ぶ人・開発する人の両方に役立つよう構成されています。 各章は短い文章でゆっくり説明しており、難しい言葉には補足や図を入れています。

使い方のコツ

  • まずは 第2章 を読んでゲームの全体像をつかみましょう。
  • 遊ぶだけなら、第5章と第6章を読めば十分です。
  • カスタマイズしたい人は、第7章と第8章をじっくり読んでください。
  • JavaScript のコードを書き換える前に、第9章と第11章を確認しましょう。

2. ゲーム全体の概要

2.1 Yuローグライクとは

Yuローグライクは、ダンジョンを選んで冒険するブラウザゲームです。HTML・CSS・JavaScript だけで動くので、 Web ブラウザがあればどこでも遊べます。ゲーム内では、ダンジョンの種類や難易度を選び、登場する敵やアイテムを楽しめます。

2.2 特徴

  • シンプルな構造で、カスタマイズしやすい。
  • ミニゲームや開発者向けツールが豊富。
  • データファイルが分かれており、ゲームバランスを調整しやすい。
  • 初心者でも理解できるよう、コメントや設計書が多数用意されている。

2.3 想定するユーザー

  • ゲームを遊びたいプレイヤー
  • ダンジョンや敵を追加したい制作者
  • JavaScript を学びながらゲームを作りたい初心者
  • 教育目的で教材として使いたい人

3. 必要な環境と準備

3.1 最低限必要なもの

  • モダンブラウザ（Google Chrome, Microsoft Edge, Firefox, Safari など）
  • テキストエディター（VS Code, Atom, メモ帳でも可）
  • ゲームファイル一式（GitHub からダウンロード、または git clone）

3.2 推奨ツール

  • Git：変更履歴を記録して、いつでも元に戻せます。
  • Live Server 拡張：編集した内容をブラウザで即確認できます。
  • 画像編集ソフト：アイテムや背景画像を作りたいときに便利です。
注意: インターネットに接続できない環境でも遊べますが、外部からダウンロードした素材を使うときは、 著作権やライセンスに気をつけてください。

4. フォルダーとファイル構成

プロジェクトの中身を理解すると、どこを編集すればよいかがすぐわかります。 下の表は代表的なファイルの場所と役割です。

場所 内容 初心者向けメモ
index.html ゲーム画面の土台。HTML のレイアウトを定義しています。 ボタンやカードの配置を変えたいときに使います。
style.css 見た目（色・フォント・余白など）をまとめています。 色を変えたいときはこのファイルを編集します。
main.js ゲームのメインロジック。タブ切り替えやボタン操作を制御します。 挙動を変えるときは慎重に。小さく編集しながら試しましょう。
js/ 機能ごとの JavaScript ファイルが入っています。 処理を分けたいときに便利です。
games/ ミニゲームなどの追加コンテンツの情報が入ります。 新しいゲームを入れたいときに使います。
dungeontypes/ ダンジョンの種類に関するデータファイル群です。 敵の強さや報酬を調整できます。
blockdata.js / blockdata.json ブロック次元モード用のデータ。複雑な設定が入っています。 最初は blockdata.json を読み、形を把握しましょう。
tools/ 制作者向けのツールやテンプレートがあります。 自分用にアレンジして使えます。

5. はじめての起動

5.1 ダウンロードと起動手順

  1. プロジェクトのフォルダーをダウンロードします。
  2. index.html をダブルクリックしてブラウザで開きます。
  3. ゲームのタイトル画面が表示されたら準備完了です。

5.2 Live Server を使う場合

  1. VS Code を開き、左側の拡張機能から「Live Server」をインストールします。
  2. フォルダーを開いた状態で、右下の「Go Live」ボタンを押します。
  3. 自動的にブラウザが開き、変更があるたびにページが更新されます。
困ったときは
  • 画面が真っ白なときは、ブラウザの開発者ツール（F12）でエラーを確認。
  • 保存し忘れると変更が反映されません。編集後は必ず保存しましょう。

6. 画面と操作の解説

6.1 タイトル画面

画面上部にタイトルと大きなカードが表示されます。カード内で、難易度・世界・ダンジョンを順番に選びます。

6.2 タブの役割

  • 通常：一般的なダンジョンを選ぶ場所です。
  • ブロック次元：複数のブロックを組み合わせてダンジョンを作ります。
  • ミニゲー経験：過去に遊んだミニゲームから得た要素を確認できます。
  • ツールズ：開発者向けの補助ツールにアクセスします。

6.3 ダンジョン詳細カード

世界とダンジョンを選ぶと、右側に詳細カードが表示されます。ここでは、ダメージ倍率・推奨レベル・説明が確認できます。 気に入ったら「ダンジョンに入る」ボタンを押しましょう。

6.4 ブロック次元モードの画面

ブロック次元タブでは、NESTED 値を入力し、複数のリストからブロックを選びます。選択内容は下部のプレビューに反映されます。 ランダム選択ボタンもあるので、迷ったら押してみましょう。

7. データファイルの内容

7.1 blockdata.json / blockdata.js

ブロック次元で利用する設定がまとまっています。JSON 形式で書かれているため、項目名と値のペアで管理します。 迷ったときは、blockdata.json をエディターで開き、どんな構造かを見てください。

主なフィールド

  • name：ブロックの名前
  • level：推奨レベル
  • type：カテゴリ（例：洞窟、塔など）
  • size：広さや階層の情報
  • chest：宝箱の数やレア度

7.2 dungeontypes フォルダー

世界やダンジョンを分類するファイルがあります。ゲーム内に出てくる説明文やアイコンなどもここで管理されます。 新しくダンジョンを追加したい場合は、既存ファイルを参考にして同じ形式で書きましょう。

7.3 games フォルダー

ミニゲームの設定やデータがまとまっています。ミニゲームを追加するときは、ここに新しいファイルを作ります。

バックアップ推奨: データファイルを編集する前に必ずコピーを作り、失敗しても戻せるようにしておきましょう。

8. コンテンツの追加・変更

8.1 ダンジョンを追加する手順

  1. dungeontypes フォルダーに新しい JSON ファイルを作る。
  2. 既存のファイルを参考に、name・description・level などを書き込む。
  3. main.js の読み込み処理に新しいファイルを追加する。
  4. ブラウザを更新して、リストに表示されるか確認する。

8.2 難易度を調整する

難易度はダメージ倍率や敵の強さに関係します。調整したいときは、該当するダンジョンのデータを変更し、プレイして確認しましょう。

8.3 見た目を変える

  • style.css の色やフォントを変更する。
  • 背景画像を追加したい場合は、CSS に background-image を設定する。
  • ボタンの形を変えたいときは、border-radius や box-shadow を調整する。
少しずつ試す：大きな変更を一度に行うと原因が分かりにくくなります。 1つの項目を変えたら保存し、ブラウザで結果を確認しましょう。

9. JavaScript の仕組み

9.1 main.js の役割

main.js は、ページの読み込み後にタブやボタンを動かすためのスクリプトです。 DOM（HTML の部品）を取得し、クリックイベントに処理を割り当てます。

9.2 データの読み込み

fetch API を使って JSON ファイルを読み込み、ダンジョンリストを作成します。Promise の仕組みが分からない場合は、 まずは async/await の基本を学んでから触ると理解しやすいです。

9.3 よく使う関数

  • renderWorldButtons()：世界のボタンを作成。
  • renderDungeonDetails()：選択したダンジョンの情報をカードに表示。
  • setupTabs()：タブ切り替えの動きを初期化。

9.4 コードを読む順番

  1. main.js の冒頭で定義されている定数を確認。
  2. イベントリスナーの登録部分を読む。
  3. 呼び出されている関数の中身を追っていく。

10. ミニゲームとツールズ

10.1 ミニゲームの扱い

プロジェクトには、別のミニゲームやアイデアノートが多数含まれています。 games フォルダーにはサンプルがあるので、自分のゲームを追加するときの参考になります。

10.2 ツールズタブ

ツールズタブからは、ブロックデータ編集など、制作時に役立つユーティリティにアクセスできます。 まだ完成していないツールもあるため、更新履歴をチェックして進捗を把握しましょう。

10.3 連携のヒント

  • ミニゲームの結果を本編に反映させる場合は、共通のデータ管理ファイルを作ると便利です。
  • UI を統一したいときは、共通の CSS クラスを用意しましょう。

10.4 ミニゲームの遊び方

ここでは、収録されているミニゲームやユーティリティの遊び方・使い方を順番にまとめています。 すべてブラウザだけで動作し、基本操作はキーボードかマウスだけで完結します。 初めて触るときは「目的」「操作」「コツ」を確認してからプレイするとスムーズです。

2048 パズル

  • 目的： 同じ数字ブロックを重ねて 2048 以上のタイルを作ります。
  • 操作： 矢印キーで全ブロックを同時にスライドさせます。
  • コツ： 角に大きな数字を集め、行を崩さないように意識しましょう。

AIM トレーナー

  • 目的： ランダムに出現するターゲットを素早くクリックし、反射神経を鍛えます。
  • 操作： マウスクリックでターゲットを消します。
  • コツ： ポインターを中央に構え、リズムよくクリックすると高得点を狙えます。

ブロック崩し（マウス操作）

  • 目的： ボールを跳ね返して全ブロックを壊します。
  • 操作： マウスでパドルを左右に動かしてボールを弾きます。
  • コツ： 角度を調整してブロックの隙間に入り込ませると一気に崩せます。

ブロック崩し（キーボード操作）

  • 目的： 矢印キーでパドルを操作し、ブロックを破壊します。
  • 操作： 左右キーで移動、スペースでゲーム開始です。
  • コツ： 連続でボールを落とさないよう、常に次の反射位置を予測しましょう。

バブルシューター

  • 目的： 同じ色のバブルを 3 個以上つなげて消し、画面上部を整理します。
  • 操作： マウスで狙いを定め、クリックでバブルを発射します。
  • コツ： 壁に反射させて奥を狙うと連鎖が作りやすくなります。

電卓ユーティリティ

  • 目的： 四則演算を素早く計算する簡易電卓です。
  • 操作： ボタンをクリック、またはキーボードで数字・演算子を入力します。
  • コツ： クリアボタンとバックスペースの使い分けでミスを減らせます。

チェッカー

  • 目的： 相手の駒を飛び越えて取りつつ、最後まで生き残ります。
  • 操作： 駒をクリックして移動先を選びます。取りができるときは優先です。
  • コツ： 端に駒を残し、キング（裏返し駒）を増やす戦術が有効です。

チェス

  • 目的： キングをチェックメイトして勝利します。
  • 操作： 駒をクリックし、有効なマスへ移動させます。
  • コツ： 序盤は中央を支配し、キングの安全を確保してから攻めましょう。

Clock Hub

  • 目的： 世界時計やアラームをまとめて確認できる時間管理ツールです。
  • 操作： 表示したい都市を選ぶか、アラームを入力して保存します。
  • コツ： ゲーム開発のテスト時間を設定しておくと進行管理に便利です。

カウンター

  • 目的： ボタンを押すたびに数値を増やし、記録や練習回数を測ります。
  • 操作： 増減ボタンをクリック。リセットで 0 に戻ります。
  • コツ： キーボードショートカットに慣れると素早くカウントできます。

ダイアグラムメーカー

  • 目的： 図やフローチャートをブラウザ上で簡単に作成するツールです。
  • 操作： ノードを追加し、ドラッグで配置、線を引いて関係を表します。
  • コツ： テンプレートを活用し、ミニゲームの設計図を素早く共有しましょう。

Dino Runner

  • 目的： 走る恐竜を障害物に当てずに走らせ続け、高スコアを目指します。
  • 操作： スペースか上キーでジャンプ、下キーでしゃがみます。
  • コツ： 障害物の高さを覚えてジャンプのタイミングを一定に保ちましょう。

ドッジレース

  • 目的： 前方から来る障害物を避けながら、できるだけ長く走行します。
  • 操作： 矢印キーで左右へ移動します。
  • コツ： 障害物の並びを予測し、早めにラインを変更すると安全です。

電子楽器

  • 目的： 仮想鍵盤を鳴らして音を確認するミニシンセです。
  • 操作： キーボードやクリックで音階を鳴らします。
  • コツ： 和音チェックや効果音の確認に使うと制作がはかどります。

表計算エクセラー

  • 目的： 簡易スプレッドシートで数値を整理・計算します。
  • 操作： セルをクリックして値や式を入力します。
  • コツ： ダンジョンデータの調整メモに使うと変更履歴を記録しやすいです。

フォーリングシューター

  • 目的： 落ちてくる敵を撃ち落として得点を稼ぎます。
  • 操作： 矢印キーで移動し、スペースでショットを発射します。
  • コツ： 敵の降下速度を見て、移動と射撃を同時に行いましょう。

フラッピーバード風

  • 目的： パイプの間を通り抜けながら飛行距離を伸ばします。
  • 操作： スペースかクリックで羽ばたきます。
  • コツ： 長押しではなく小刻みにタップし、高さを細かく調整します。

ギャンブルホール

  • 目的： シンプルな賭け遊びでポイントを増やすミニゲーム集です。
  • 操作： ベット額や選択肢をクリックして結果を確認します。
  • コツ： 所持ポイントが尽きないよう、小さな額から試しましょう。

囲碁ライト

  • 目的： 石を置いて陣地を広げる囲碁の簡易版です。
  • 操作： 交点をクリックして石を配置します。
  • コツ： 呼吸点を意識し、相手の石を囲む形を作りましょう。

インベーダー風シューティング

  • 目的： 上から降りてくる宇宙人を倒し、全波をクリアします。
  • 操作： 左右キーで移動、スペースでショットを撃ちます。
  • コツ： 障害物を盾にしつつ、横列を一列ずつ片付けるのが安全です。

算数ラボ

  • 目的： 計算問題を解いて基礎力を高める学習向けコンテンツです。
  • 操作： 表示された問題に解答を入力し、送信ボタンで採点します。
  • コツ： 制限時間がある場合は暗算とメモを併用してスピードを上げましょう。

マッチ3 パズル

  • 目的： 同じアイコンを 3 つ以上揃えて消し、目標スコアを達成します。
  • 操作： 隣り合うピースをドラッグまたはクリックで入れ替えます。
  • コツ： 連鎖を狙える形を優先して入れ替えましょう。

マインスイーパー

  • 目的： 爆弾を踏まずに全ての安全なマスを開きます。
  • 操作： 左クリックで開く、右クリックで旗を立てます。
  • コツ： 数字が示す周囲の爆弾数をもとに論理的に開けていきましょう。

音楽プレイヤー

  • 目的： BGM や効果音をチェックできる再生ツールです。
  • 操作： プレイリストから曲を選んで再生、シークバーで位置を調整します。
  • コツ： ループ再生を活用し、音量バランスを事前に確認しましょう。

メモ帳ユーティリティ

  • 目的： テキストメモを簡単に書き留め、保存します。
  • 操作： 入力エリアに自由に書き込み、保存ボタンでローカルに保持します。
  • コツ： 開発タスクやテスト結果をすぐ記録できるよう開いたままにしておきましょう。

オセロ

  • 目的： 石を挟んで色を反転させ、最後に多数を確保します。
  • 操作： マスをクリックして石を置きます。
  • コツ： 隅を取ると盤面を安定させやすいです。

パックマン風

  • 目的： 迷路内のドットを全て食べ、ゴーストを避けます。
  • 操作： 矢印キーで移動します。
  • コツ： パワークッキーを取るタイミングを合わせ、追跡から逃れましょう。

ペイントツール

  • 目的： 簡単なドット絵や図を描くツールです。
  • 操作： キャンバス上をドラッグして描画し、色や太さを選びます。
  • コツ： レイヤーがないので、こまめに保存しながら描きましょう。

物理サンドボックス

  • 目的： 物理演算の動きを確認できるデモです。
  • 操作： オブジェクトを追加したり、重力や衝突設定を変更します。
  • コツ： ゲームで使いたい動きを試作し、値をメモしておきましょう。

ポン

  • 目的： ボールを打ち返し、相手ゴールに入れて得点します。
  • 操作： 矢印キーまたは W/S キーでパドルを上下に動かします。
  • コツ： ボールが当たる位置で反射角度が変わるので、狙って打ち返しましょう。

ポモドーロタイマー

  • 目的： 25 分作業 + 5 分休憩のサイクルで集中力を保つ時間管理ツールです。
  • 操作： スタートボタンで計測を開始し、アラートに従って休憩します。
  • コツ： マニュアルを読みながら開発する際の時間配分に便利です。

ぷよぷよ風

  • 目的： 同じ色のぷよを 4 つ以上繋げて消し、連鎖を狙います。
  • 操作： 矢印キーで移動・回転、下キーで落下速度を上げます。
  • コツ： 階段・カギ積みなど定番の積み方を練習すると連鎖が作りやすいです。

川渡りパズル

  • 目的： 全員を安全に対岸へ運ぶ論理パズルです。
  • 操作： 乗せるメンバーを選び、渡るボタンで移動させます。
  • コツ： 禁止条件を常に意識し、片道ずつ安全性を確認しましょう。

さめがめ

  • 目的： 同じ色が繋がったブロックを消して高得点を目指します。
  • 操作： グループをクリックするとまとめて消えます。
  • コツ： 大きな塊ができるまで待ち、最後に連鎖させるとスコアが伸びます。

スライディングパズル

  • 目的： バラバラのパネルを正しい順番に並べ替えます。
  • 操作： 空白と隣接するピースをクリックして入れ替えます。
  • コツ： 上段・左列から順に揃え、最後の 2×2 を回して完成させます。

スネーク

  • 目的： エサを食べて体を伸ばし、壁や自分にぶつからずに記録を更新します。
  • 操作： 矢印キーで進行方向を変えます。
  • コツ： 余裕をもって曲がり、体が長くなったらジグザグで安全地帯を作りましょう。

手先のワイヤーゲーム

  • 目的： ワイヤーから外れないようにカーソルを動かす集中力ゲームです。
  • 操作： マウスでラインに沿って移動します。
  • コツ： 細かい動きが必要なので、マウス感度を低めにすると安定します。

石盤ボードゲーム集

  • 目的： 囲碁や五目並べなど石を使うボードゲームのサンプル集です。
  • 操作： ルールに応じてマスをクリックし、石を配置します。
  • コツ： ゲームごとのルールメモを見ながら遊ぶと混乱しません。

ストップウォッチ

  • 目的： 経過時間を測る基本ツールです。
  • 操作： スタート・ストップ・リセットの各ボタンをクリックします。
  • コツ： タイムアタックの記録に活用すると改善点が分かります。

数独

  • 目的： 9×9 のマスに 1〜9 の数字を重複なく配置します。
  • 操作： マスをクリックし、数字を入力します。
  • コツ： 行・列・ブロックごとに候補をメモし、確定情報を順番に埋めていきます。

テンテンパズル

  • 目的： 10 を作るように数字を組み合わせてボードを整理します。
  • 操作： 数字を選んで配置し、ラインが揃うと消えます。
  • コツ： 余りが出ないように数字ブロックの形を確認してから置きましょう。

テスターツール

  • 目的： ミニゲームの動作確認を素早く行うためのチェック用コンポーネントです。
  • 操作： テスト項目を選び、想定ケースを実行して挙動を確認します。
  • コツ： 新機能を追加したら必ずテスターで確認し、バグを早期発見しましょう。

テトリス風

  • 目的： ブロックを積み上げずにラインを消し、スコアを伸ばします。
  • 操作： 矢印で移動・回転、下でソフトドロップ、スペースでハードドロップ。
  • コツ： ホールド機能がある場合は使いどころを決め、T スピンなどのテクを練習しましょう。

タイマー

  • 目的： 指定時間のカウントダウンを行うシンプルなツールです。
  • 操作： 分・秒を設定してスタートします。
  • コツ： 開発の休憩やイベントの残り時間確認に活用できます。

ToDo リスト

  • 目的： やることを整理し、進捗を管理します。
  • 操作： タスクを入力して追加、完了したらチェックを入れます。
  • コツ： 細かい作業に分解して登録すると達成感が得られやすいです。

トリオミノコラム

  • 目的： 3 つ組ブロックを落として列を揃える落ち物パズルです。
  • 操作： 矢印キーで移動・回転させ、落下位置を決めます。
  • コツ： 色の並び替えを意識し、同色列を作って消去を狙いましょう。

トランプゲーム集

  • 目的： ソリティアなどのトランプ遊びをまとめたコレクションです。
  • 操作： ルールに沿ってカードをドラッグ&ドロップします。
  • コツ： 各モードのヘルプを確認し、配置手順を覚えるとスムーズです。

アルティメット三目並べ

  • 目的： 3×3 の盤面それぞれが小さな三目並べになった上級ルールです。
  • 操作： 指定された小盤にマークを置きます。
  • コツ： 次に相手が打つ盤面を読んで、大盤のラインを優先しましょう。

動画プレイヤー

  • 目的： チュートリアル動画やリファレンス映像を再生するツールです。
  • 操作： 動画を選び、再生ボタンやシークバーで操作します。
  • コツ： 倍速再生を活用して学習効率を上げましょう。

モグラたたき

  • 目的： 穴から出るモグラをタイミングよく叩いて得点を稼ぎます。
  • 操作： 出現したモグラをクリックします。
  • コツ： 出現パターンを覚え、視線を動かし過ぎないのがポイントです。

ワーディングツール

  • 目的： 文章を整えたり、語句候補をまとめるライティング支援ツールです。
  • 操作： テキストを入力し、候補の提案や置き換えを確認します。
  • コツ： ミニゲームの説明文やイベントテキストを作るときに活用すると便利です。

11. 開発フローとヒント

11.1 作業の基本サイクル

  1. やりたいことをメモする。
  2. 対象のファイルをバックアップする。
  3. 小さく変更し、ブラウザで確認する。
  4. 動作が安定したら、変更内容をメモや Git に記録する。

11.2 Git の簡単な使い方

  • git status：変更点を確認。
  • git add .：変更をステージに載せる。
  • git commit -m "メモ"：変更を記録する。
  • git log：過去の履歴を見る。

11.3 チーム開発するときのポイント

  • 役割を分担し、作業中のファイルを共有する。
  • 変更内容を README やマニュアルに追記しておく。
  • 画面キャプチャを撮って変更点を視覚的に伝える。

12. よくある質問

Q1. 画面が正しく表示されません。

ブラウザのキャッシュが残っているかもしれません。ページをリロード（Windows: Ctrl + F5、Mac: Cmd + Shift + R）してみましょう。

Q2. データを変更したらエラーが出ました。

JSON 形式のカンマ漏れや引用符の閉じ忘れが多いです。エディターの構文チェック機能を使うと発見しやすくなります。

Q3. モバイルで遊べますか？

はい、レスポンシブデザインに対応しているのでスマートフォンでも遊べます。ただし、画面が小さいため横向きにすると操作しやすいです。

13. 用語集

DOM
ブラウザが理解できる HTML の部品の木構造。JavaScript で操作します。
JSON
データを「キー：値」で表す形式。人もコンピューターも読みやすいのが特徴です。
レスポンシブデザイン
画面の幅によってデザインを自動で調整する仕組み。スマホやタブレットで活躍します。
タブUI
複数の画面を切り替えるためのメニュー。クリックすると内容が変わります。
